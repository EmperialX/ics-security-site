Understanding IoT and Security Problems

IoT devices use sensors to transmit and receive data over the Internet and can sync with various devices. Due to the lack of regular security updates, these devices are highly vulnerable to malware attacks, allowing attackers to turn them into botnets for launching cyberattacks.

IoT devices often lack proper authentication, use default credentials, and lack strong encryption or key management systems, making them vulnerable to various types of attacks.

What is IoT?

IoT, or Internet of Everything (IoE), refers to web-enabled computing devices that can sense, collect, and send data using embedded sensors, communication hardware, and processors.

An IoT "thing" is a device implanted on natural, man-made, or machine-made objects with network communication functionality.

IoT uses existing and emerging technologies for sensing, networking, and robotics, enabling deeper analysis, automation, and integration. Key features include:

  • Connectivity
  • Sensors and actuators
  • Artificial intelligence
  • Small, embedded devices
  • Active engagement with environment

IoT Architecture

The IoT architecture comprises several layers:

  • Edge Technology Layer: Includes hardware like sensors and RFID tags for data collection.
  • Access Gateway Layer: Bridges the gap between devices and clients, handling message routing and identification.
  • Internet Layer: Enables communication between devices, clouds, and local gateway services.
  • Middleware Layer: Acts as an interface between the application and hardware layers, managing data and devices.
  • Application Layer: Delivers services to users in sectors like building, industry, and healthcare.

IoT Security Problems

IoT devices often lack proper authentication, use default credentials, and lack strong encryption or key management systems. Key areas of vulnerability include:

  • Application: Input validation, authentication, authorization, lack of automatic security updates, and default passwords.
  • Network: Firewall issues, improper encryption, and lack of automatic updates.
  • Mobile: Insecure APIs, lack of communication channel encryption, and lack of storage security.
  • Cloud: Improper authentication and lack of encryption for storage and communications.

The security equation: Application + Network + Mobile + Cloud = IoT

Recognizing Different Types of IoT Threats

IoT threats such as distributed denial-of-service (DDoS) attacks, man-in-the-middle attacks, and ransomware attacks can compromise devices and steal sensitive data from users as well as organizations.

IoT devices on the Internet have very few security protection mechanisms against various emerging threats. Attackers often exploit these poorly protected devices to cause physical damage to the network, wiretap communication, or launch disruptive attacks.

DDoS Attack

A DDoS attack involves multiple infected systems overwhelming a single online system or service, making it slow or unavailable to legitimate users.

Steps of a DDoS attack:

  1. Attacker gains remote access to vulnerable devices
  2. Malware is injected into IoT devices to turn them into botnets
  3. A command-and-control center is used to instruct botnets to send requests to the target server
  4. The target server goes offline, becoming unavailable

In late 2017, the Satori botnet infected over 260,000 home routers within just 12 hours by exploiting default usernames and passwords in D-Link DSL routers.

Attack on HVAC Systems

Attackers exploit vulnerabilities in IoT-enabled heating, ventilation, and air conditioning (HVAC) systems to steal login credentials and gain unauthorized access to networks.

Steps to exploit HVAC systems:

  1. Attackers search for vulnerable ICS systems using tools like Shodan
  2. Attackers attempt to use default user credentials with online tools
  3. Remote access to the HVAC system is attempted through the ICS system
  4. Once accessed, the attacker can control the temperature or launch further attacks on the local network

Tools used by attackers include Shodan (https://www.shodan.io) to search for vulnerable systems and online default credential databases (https://madifi.de/defpass/index.php).

IoT Forensics

IoT forensics involves the investigation of IoT devices and their connected devices. Investigators need to understand forensic tools, data acquisition, and examination procedures.

IoT forensics is a multidisciplinary approach within digital forensics that addresses cybersecurity threats targeting IoT devices, focusing on the identification, acquisition, and analysis of digital evidence from IoT devices, sensors, and nodes.

To conduct a forensic investigation, investigators should consider technological aspects across three zones:

  • IoT Zone: The devices and their embedded software
  • Network Zone: Communication protocols and traffic
  • Cloud Zone: Data stored and processed in cloud services

IoT Forensics Process

The IoT forensic process involves several key steps:

  • Evidence Identification and Collection: Identifying affected IoT devices at the crime scene to increase the scope of the investigation
  • Preservation: Ensuring the evidence is properly preserved, which may involve avoiding switching off the device before data acquisition
  • Data Analysis: Analyzing data stored on cloud services or mobile phones synced with IoT devices
  • Presentation and Reporting: Preparing a report on the evidence and forensic analysis

Key challenges include device identification due to their autonomous nature, ensuring evidence integrity, determining if malfunctions are due to human intervention or design flaws, and proper documentation of findings.

Performing Forensics: Android Wear

Smartwatches connect to smartphones or networks via Bluetooth, Wi-Fi, GPS, and NFC, primarily for tracking exercises, checking time, messages, and emails.

Network connections include NFC for short-range wireless connections, Bluetooth for pairing with other devices, Wi-Fi for wireless networking, and GPS for location tracking. Hackers can exploit motion sensors and GPS systems to gather sensitive user information, such as PINs, passwords, and physical location.

Logical Acquisition:

  1. Enable developer options
  2. Connect to Wi-Fi
  3. Enable Wi-Fi debugging
  4. Obtain IP address
  5. Use ADB to pull data

Physical Acquisition:

  1. Establish connection with SDK tools
  2. Execute ADB commands
  3. Verify connection
  4. Generate partition image
  5. Requires rooted device

Hardware Level Analysis

When software-level access to a device's physical contents is difficult, hardware-level acquisition methods can be used:

JTAG Forensics:

Allows access to the physical contents of devices to acquire a physical image of the device's memory chip.

JTAG (Joint Test Action Group) method provides access to test ports on electronic devices to extract data at the hardware level.

Tools required:

  • Equipment to dismantle the device
  • Wiring and soldering equipment
  • JTAG software and hardware

Chip-Off Forensics:

Involves carefully extracting the memory chip from the device's motherboard to read it with chip-reading equipment.

This is a destructive method that may result in data loss, and the removed memory chip cannot be re-fixed for reuse.

Equipment needed:

  • Memory chip reading hardware
  • Adapter to hold the chip
  • Forensic workstation software

Module Summary

Internet of Things (IoT): Refers to physical devices embedded with sensors, software, and other technologies to collect and share data with connected devices and systems via the internet.

IoT devices are increasingly common and are vulnerable to cyber attacks. Common threats include:

  • DDoS attacks - Multiple infected systems overwhelming a target
  • Rolling Code attacks - Targeting smart locking systems in vehicles
  • BlueBorne attacks - Exploiting Bluetooth connections
  • Jamming attacks - Disrupting wireless communications
  • Ransomware attacks - Encrypting user data for ransom
  • Attacks using backdoors - Gaining unauthorized access

Forensic investigators should understand device technologies, acquisition procedures, and examination tools. In cases where the device is damaged or data acquisition through software is not possible, JTAG and Chip-off forensics can be used to acquire a physical image of the device.
